harnes v0

Research wrapper around local LLMs
Meta-cycle over ReAct для универсальных автономных агентов
2026-05-26 · статус + roadmap

Проблема

Универсальный автономный агент — конъюнкция трёх свойств, по отдельности достигнутых, но не вместе.

Объект

  • универсальный — относительно распределения задач (held-out)
  • автономный — низкий локус и крупная зернистость контроля
  • агент — действует на среду, не просто предсказывает

Пробел

  • Фундаментальные модели: общность без действия
  • RPA: автономность без общности
  • Боты: агентность без общности и надёжности
  • Открыт вопрос их конъюнкции

Две измеримые оси

  • Ширина распределения — оценка на не виданных при разработке средах
  • Длина горизонта — компетентность во времени, нелинейная деградация (компаундинг ошибки)

Критерий прогресса — производительность на отложенном распределении при ограниченном бюджете вмешательств. Надёжность — first-class метрика.

Гипотеза

Метацикл поверх ReAct-цикла, работающий непрерывно, с явными этапами восприятия, внимания, арбитража целей, recall'а, исполнения, верификации, обновления модели мира, рефлексии и сохранения. 

[tick]
  sense                — polling + event-queue (без LLM)
  attend               — score(relevance, novelty, urgency); standing-цели
  goal_arbitration     — арбитраж + источник новых целей + pending_verifications
                         + operator-approval flow для top-level self-целей
  if active_goal:
    recall(coarse)     — единый memory layer (vector + graph + episodic)
    react_loop         — thought ↔ action ↔ observation
    verify(immediate)  — leaf-level; composite/external → pending_verifications
    world_update       — обновление явной модели мира (KG, темпоральный)
    if reflect_trigger:
      reflect          — режим выводится из триггера (batch, failure, periodic)
    store              — маршрутизация в бэкенды памяти
[end tick]

Локальная Gemma 26B-A4B (MoE, 4B активных параметров, Q4) реально:

  • Прочла цель из system-prompt
  • Сгенерировала валидный JSON action {"tool_id": "write_file", "args": ...}
  • Tool-layer провалидировал, исполнил, записал файл
  • Эмитнула {"tool_id": "finish", ...} с final_state
  • 5 шагов, 825 completion-токенов, ~9 секунд wall-clock
  • Trajectory сохранена в LanceDB; goal → DONE

Что дальше — матрица value × effort

Сортировка по соотношению полезности и трудозатрат. Зелёные клетки — делать первыми; верхне-правые — research-bets, которые двигают рамку «универсальности» / «длинного горизонта».

  • Multi-tier LLM (gemma-31b-mtp для thought)
  • Standing-цели стартового набора (5–10)

Ranked: что делать в каком порядке

Top-to-bottom по value/effort. Цифры — ориентировочные.

  1. Judge predicate через LLM-судью
    ~30 LoC v0.1 разблокирует goal'ы с открытым критерием
    Сейчас verify умеет только structural. Добавим вызов анти-коррелированной модели (другой системный промпт) для judge-предикатов. Метрика «измерено LLM-судьёй» уже зашита в Verdict.measured_by.
  2. Multi-tier LLM (gemma-31b-mtp для thought, gemma-26b-a4b для attend/verify)
    ~20 LoC v0.1 конфиг + 1 функция диспетчеризации
    У нас есть 3 модели на сервере. Сейчас все запросы идут в Gemma-26B-A4B. Параметризовать llm.call(tier=...) и подменять model id.
  3. Критик на is_irreversible=true
    ~80 LoC v0.1 снимает компаундинг на необратимых шагах
    Отдельный модельный вызов с «ролью критика» перед action.is_irreversible=true. Verdict из CritiqueStep уже определён в схеме; нужно только wired в loop.py.
  4. Standing-цели стартового набора
    ~100 LoC + YAML v0.1 даёт «характер» агенту
    5–10 политик: «при verify=fail с приоритетом≥P → inquiry: диагностика», «через N тиков без цели → inquiry: что делать», «episodic-лог > M → maintenance».
  5. Reflect в режиме failure_analysis
    ~200 LoC v0.2 первый контур самоулучшения
    На каждый verify=fail — reflect режима failure_analysis, выход — обновлённая версия скилла (через SkillRegistry.save). Версионирование через git-тег. Метрики per-version уже копятся.
  6. Real Graphiti + Neo4j wiring (world_update)
    ~300 LoC + async v0.2 оживляет world model
    Заменить stub в memory/world.py на реальный graphiti_core. Async-обёртка asyncio.run(...) для синхронного метацикла. Поднимать Neo4j в docker-compose (уже в файле).
  7. MemoryAgentBench реальный adapter
    ~500 LoC v0.2 разблокирует sci-сопоставление
    Реализовать BenchmarkAdapter Protocol для MemoryAgentBench (ICLR 2026). Перевести их задачи в наши Goal'ы, верификация по их dataset-specific критериям. Это даёт первые числа сравнимые с литературой.
  8. Self-generated goals — curiosity + maintenance
    ~400 LoC v1.0 ключ к автономности
    В goal_arbitration добавить модуль «гипотеза» — порождает inquiry-цели из novelty в attend (curiosity) и из метрик памяти (maintenance: consolidate, prune). Верхне-уровневые self-цели идут через operator-approval, уже реализовано.
  9. Held-out evaluation set construction
    manual + ~200 LoC v1.0 точка фальсифицируемости
    Собрать набор задач из открытых распределений (не из MemoryAgentBench), на которых разработка не велась. Это и есть «held-out» в твоём критерии прогресса. Без него заявление об общности нефальсифицируемо.

Roadmap по milestones

MilestoneЧто доказываетСостав
v0 done Архитектура работоспособна. Локальный LLM умеет ReAct. 14 задач вертикального среза. 124 теста. E2E с реальной моделью.

Ось измерения

На каждом milestone — прогон полного eval-сета. Метрики: success-rate на held-out, среднее число human-interventions на единицу работы, длина горизонта без коррекции, failure-mode breakdown.

Это даёт численный ответ на «ширина распределения × длина горизонта» в духе твоей рамки.
