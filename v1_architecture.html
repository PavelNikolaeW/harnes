02 · framing research

Зачем

Универсальный автономный агент на открытом распределении задач, длинном горизонте, с надёжностью как first-class метрикой.

Пять осей

  • Универсальность — широкое распределение задач
  • Автономность — степень self-direction
  • Агентность — длительная целенаправленность
  • Открытость — данные не из training/dev
  • Горизонт — multi-step / multi-session

Главный артефакт

Не «продукт», а research-инструмент: каждое улучшение должно быть измеримо на held-out наборе с reliability метриками.

В v1.0 это материализовалось как

  • #31 — held-out flag
  • #32 — pass@k, p95, entropy
  • #33 — self-generated goals
  • #34 — recall first-class
  • #35 — journal + recovery

08 · memory storage

Memory-слой

Четыре backend'а под одним MemoryRouter.recall(query, types, k).

episodic

LanceDB (file-based vector store)

Trajectory steps. recent_steps(limit) + ANN. Каждый шаг ReAct тут — на нём reflect анализирует прошлое.

semantic

Qdrant (vector DB)

Pre-encoded facts. Embedding-based поиск. В v0 не наполняется, готов к use.

world

Graphiti + Neo4j (temporal KG)

add_episode при store-стейдже. search(query, valid_at) возвращает узлы с свойствами.

MemoryRouter

Координатор. На каждом recall — параллельный запрос к доступным backends, агрегация в MemoryBundle.

class MemoryBundle:
    episodic: list[EpisodicRecord]
    semantic: list[SemanticRecord]
    world: list[WorldNode]

router.recall(query, types=None, k=10)  # None = all available

Болевые точки решены

  • Graphiti default OpenAIClient использует responses.parse — не работает с ll-router → переключились на OpenAIGenericClient (chat.completions)
  • Neo4j driver 15s+ timeout → 2s TCP precheck (_bolt_reachable) + broken-cache
  • asyncio.run() ломает Graphiti driver → persistent event loop
09 · tools capability

Tool-слой

Pipeline (registry.invoke)

raw_args → validate(input_schema) → resolve_irreversibility
invoke(impl) → classify(outcome) → retry? → validate(output_schema) → ObservationStep

7 outcomes: success / tool_error / schema_error / timeout / permission_denied / malformed_output / semantic_error

Irreversibility

3 уровня: NEVER / CONDITIONAL / ALWAYS. Skill может override через irreversibility_overrides[tool_id]. Conditional — через registered predicate (@register_predicate).

Тулы с is_irreversible=True идут через критик в react/critic.py перед исполнением.

build_runtime_registry(router) — v1.0 #34

12 · react inner loop

ReAct internal loop

Step types (discriminated union)

ThoughtStep PlanStep ActionStep ObservationStep CritiqueStep RetryNoteStep

run_react (Classic)

for step in range(max_steps):
    if total_tokens >= budget_tokens:
        traj.status = BUDGET_EXCEEDED; break

    thought = _thought_call(system_prompt, traj, llm)   # tier=main
    action  = _action_call(system_prompt, traj, llm)
    if action is None:                  # parse failed
        append MALFORMED_OUTPUT obs; continue

    action.is_irreversible = registry.resolve_irreversibility(...)

    if action.tool_id == "finish":
        traj.status = SUCCESS; traj.final_state = action.args; break

    if with_critic and action.is_irreversible:
        critique = critique_action(action, traj, goal)   # tier=light
        if critique.verdict == REJECT:
            critic_rejections += 1
            if critic_rejections > max: traj.status = FAILURE; break
            continue   # skip execute, retry thought

    obs = registry.invoke(action.tool_id, action.args, skill=skill)
    if _detect_loop(traj): traj.status = FAILURE; break
else:
    traj.status = FAILURE   # max_steps hit

Trajectory несёт started_at / ended_at / total_cost + полный список Step'ов. Это сериализуется в episodic и читается reflect'ом.

v1.0 #33 — inquiry from failure

Новый второй источник self-spawn. На verify=FAIL reflect-LLM анализирует траекторию и решает, есть ли concrete knowledge-gap, который стоит закрыть отдельным inquiry.

inquiry = Goal(
    description="Find out how X works",
    goal_class=GoalClass.INQUIRY,
    origin=Origin.SELF,
    origin_subtype=OriginSubtype.LEARNING,
    parent_id=goal.id,
)

Falls open: при ошибках LLM → None, тик не падает.

Self-generation rate

В run-loop CLI считается:

self_gen_rate = ticks_with_self_spawn / total_ticks
  → "Self-generated: 12 goals in 8/47 ticks (17.0% self_generation_rate)"

Это фундаментальная метрика автономии — насколько часто агент сам себе ставит цели без оператора.

15 · reflect self-improve

Reflect — два режима на FAIL

failure_analysis (v0.2)

LLM (tier=main) анализирует:

  • Что пошло не так?
  • Поможет ли изменение prompt_template?
  • Если да — full updated template.

SkillRegistry.save новой версии с bump patch (0.0.1 → 0.0.2). Старая остаётся, parent_version_id зафиксирован. Git auto-commit (если включён) делает атомарный коммит.

Sanity: {goal_description} и {tools_list} placeholders должны сохраниться.

inquiry_from_failure (v1.0 #33)

LLM решает: «есть ли в провале concrete knowledge-gap?» (factual / API / concept).

Консервативно — should_spawn_inquiry должен быть явный true, description ≥ 8 chars, иначе None.

Запускается параллельно с failure_analysis — два независимых выхода: bump skill prompt + spawn inquiry-goal. Падение одного не блокирует другой.

Когда триггерится

if verdict.status == FAIL and trajectory and active_goal:
    _maybe_reflect_failure(state, skill_registry, goal_repo)
    # → возможный bump skill + возможный spawn inquiry

Recovery

На старте run-loop --resume читает latest_snapshot() и продолжает с tick_id+1, сохраняя счётчики.

Falls open

  • Append/snapshot exception → log warning, loop продолжается
  • _build_url: read-only FS → fallback :memory:
  • Падение одного тика → ERROR event, continue (не валит loop)
  • Финальный snapshot в finally блок — даже на Ctrl+C
