Standing-цели — реактивный слой. Они не действуют сами, только триггерят подцели на основе сигналов окружения / истории (alert observed, prev verify failed, deadline approaching). Их статус стабильно ACTIVE; они не закрываются.

Нормальный размер набора — 5–15 standing-policy. Это формирует «характер» агента: что он считает достойным внимания. Слишком много → шум и постоянный реактивный режим; слишком мало → агент глух к среде.

Маркировка spawn'а: originator = f"standing:{parent.id}" (см. harnes/metacycle/standing.py). Effectiveness = done_children / spawned; «gone» = child удалён из GoalRepository (clean run / retention), в done не учитывается.
